= 错误代码
:page-toclevels: 1

如果某个特定的原生 API 调用失败，Neutralinojs 客户端库会用一个错误对象来拒绝待处理的 Promise。

错误对象具有以下结构：

[source,javascript]
----
{
    code: "<code>",
    message: "<message>"
}
----

* `code` 字符串：错误代码。
* `message` 字符串：错误消息

[[native-api-error-codes]]
== 原生 API 错误代码

[%autowidth]
|===
|错误代码|    消息    |抛出位置
|`NE_FS_DIRCRER`|    无法创建目录。|    `os.createDirectory`
|`NE_FS_REMVERR`|    无法删除路径。|    `os.remove`
|`NE_OS_TRAYIER`|    无法初始化托盘菜单。此错误通常在 GNU/Linux 系统中不包含应用指示器库时抛出。|    `os.setTray`
|`NE_FS_FILRDER`|    文件读取错误。|    `filesystem.readFile`, `filesystem.readBinaryFile`
|`NE_FS_FILWRER`|    文件写入错误。|    `filesystem.writeFile`, `filesystem.writeBinaryFile`
|`NE_FS_FILOPER`|    文件打开错误。|    `filesystem.openFile`
|`NE_FS_UNLTOUP`|    无法更新已打开文件的 id。|    `filesystem.updateOpenedFile`
|`NE_FS_UNLTFOP`|    无法找到已打开文件的 id。|    `filesystem.getOpenedFileInfo`
|`NE_FS_NOPATHE`|    没有文件或目录。|    `filesystem.getStats`, `filesystem.readDirectory`
|`NE_FS_COPYERR`|    复制错误。|    `filesystem.copy`
|`NE_FS_MOVEERR`|    移动错误。|    `filesystem.move`
|`NE_FS_UNLCWAT`|    无法创建监视器。|    `filesystem.createWatcher`
|`NE_FS_NOWATID`|    无法找到监视器。|    `filesystem.removeWatcher`
|`NE_OS_UNLTOUP`|    由于进程标识符无效或操作不当，无法更新已启动的进程。|    `os.updateSpawnedProcess`
|`NE_OS_INVMSGA`|    无效的消息框参数。|    `os.showMessageBox`
|`NE_OS_INVKNPT`|    无效的平台路径名。|    `os.getPath`
|`NE_ST_INVSTKY`|    无效的存储键。|    `storage.getData`, `storage.setData`
|`NE_ST_STKEYWE`|    存储写入错误。|    `storage.setData`
|`NE_RT_INVTOKN`|    无效的访问令牌。|    `*`
|`NE_RT_NATPRME`|    没有权限执行提供的原生方法。|    `*`
|`NE_RT_APIPRME`|    没有权限使用原生 API。|    `*`
|`NE_RT_NATRTER`|    原生方法运行时错误。大多由于缺少参数导致。|    `*`
|`NE_RT_NATNTIM`|    原生方法未实现。|    `internal`
|`NE_CL_NSEROFF`|    无法连接到 Neutralino 服务器。|    `*`
|`NE_EX_EXTNOTC`|    扩展尚未连接|    `extensions.dispatch`
|`NE_UP_CUPDMER`|    更新清单无效或应用 ID 不匹配|    `updater.checkForUpdates`
|`NE_UP_CUPDERR`|    无法获取更新清单|    `updater.checkForUpdates`
|`NE_UP_UPDNOUF`|    未加载更新清单|    `updater.install`
|`NE_UP_UPDINER`|    更新安装错误|    `updater.install`
|`NE_RS_APIRQRF`|    只有在加载资源文件时，资源 API 才能工作|    `resources.*`
|`NE_RS_FILNOTF`|    无法在资源包中找到请求的路径|    `resources.readFile`, `resources.readBinaryFile`, `resources.extractFile`
|===
